# Seed contributor intake with labels and starter issues

## Summary
Create the first batch of labels and starter issues so the public repo can convert attention into useful contributions.

## Why
The repo now has templates and governance docs, but still needs a visible issue runway.

## Scope
- Add/use domain-specific labels
- Seed 8–12 starter issues
- Mark several as `good first issue`
- Add or link a “start here” contributor path

## Acceptance criteria
- Labels exist and are used consistently
- Starter issues are opened with clear acceptance criteria
- At least 4 issues are suitable first-contributor tasks
