# Security Policy

## Scope
If you discover a security issue in goop-veil, please report it responsibly.

This project includes areas that can affect:
- router configuration behavior
- reporting/documentation artifacts
- MCP/tool invocation surfaces
- experimental firmware paths

## How to report
Please open a private security report through GitHub security reporting if available, or contact the maintainers directly through the repository owner contact path.

When reporting, include:
- affected version / commit
- impacted component
- reproduction steps
- expected vs actual behavior
- potential impact
- any logs or artifacts (redacted as needed)

## What to expect
Target response goals:
- initial acknowledgement: within 72 hours
- severity assessment: as soon as practical after reproduction
- coordinated fix/disclosure: depends on impact and complexity

## Please avoid
- posting exploitable details publicly before maintainers have a chance to assess
- sharing secrets, credentials, or unredacted private user data in public issues

## Research preview note
This repository is currently a **software-only research preview**. Some surfaces are explicitly experimental. Please report issues against observed implementation behavior, not inferred marketing claims alone.
